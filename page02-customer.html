💡 핵심 역설: 스팸 때문에 유용한 제안도 놓친다

수신자의 가장 큰 아이러니 — 과도한 스팸 때문에 실제로 필요한 솔루션/제안도 걸러버림. 동의 기반 디렉토리는 "내가 허용한 분야의 연락만 받겠다"는 필터를 제공하여, 수신자가 필요한 정보는 받되 노이즈는 차단하는 선택권을 줍니다.

🔐 수신자가 원하는 것

✅
연락 채널 제어권 — "이메일만 허용, 전화는 불가" 같은 세밀한 설정. 시간대/빈도 제한 가능
✅
관심 분야 태깅 — "AI/ML 솔루션에만 관심", "물류 자동화만 보고 싶다" 등 카테고리별 수신 설정
✅
즉시 철회 가능한 동의 — 원클릭으로 특정 벤더 또는 전체 연락 차단. "내 데이터 삭제" 즉시 실행
✅
발신자 신뢰도 확인 — 누가 내 정보를 조회했는지, 해당 기업의 평판/리뷰 확인 후 응답 결정
✅
보상 체계 — 연락처를 공개하는 대가로 포인트/크레딧 수령. "내 데이터의 가치를 내가 받겠다"

✅ 조달 담당자가 동의 기반 디렉토리에서 원하는 것

🔍
검증된 공급업체 프로필 — 사업자등록증, 인증서, 재무제표, 신용등급이 검증 완료된 상태로 표시. 자기 선언이 아닌 제3자 인증
🔔
실시간 연락처 정확성 — 담당자가 직접 업데이트하는 연락처. 퇴사/부서 이동 시 즉시 반영. "이 번호로 전화하면 진짜 담당자가 받는다" 보장
📊
거래 이력/리뷰 — 다른 구매자들의 실제 거래 후기, 납기 준수율, 품질 평가. 익명 리뷰 + 실명 리뷰 혼합

RFQ 보안 채널 — 가격 정보가 경쟁업체에 노출되지 않는 안전한 견적 요청 채널. E2E 암호화

📊 리스크 & 과제

🔴
치킨-에그 문제 (높음) — 수신자 등록이 없으면 발신자가 오지 않고, 발신자가 없으면 수신자에게 보상할 수 없음. 초기 데이터 확보가 최대 관문. 해결: 특정 버티컬(예: IT/SaaS) 집중 론칭 후 확장
🔴
기존 업체의 모방/방어 (높음) — ZoomInfo가 "동의 기반 옵션" 추가 가능. 하지만 기존 수익 모델(스크래핑)과 충돌하여 전면 전환은 어려움 (이노베이터 딜레마)
🟡
초기 데이터 규모 (중간) — ZoomInfo 3억+ 프로필 vs 우리 초기 10만 이하. 양보다 질(정확도 95%+)로 차별화. 특정 니치에서 완전성 달성 후 확장
🟡
수신자 이탈 관리 (중간) — 스팸이 늘면 수신자 떠남. 엄격한 발신자 평판 시스템 + 월간 연락 한도 + AI 품질 필터로 방어
🟢
기술 복잡도 (낮음) — 핵심 기술은 프로필 관리 + 매칭 + 메시징. 특별한 딥테크 불필요. 프로덕트-마켓 핏과 GTM이 관건
