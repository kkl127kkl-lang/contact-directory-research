화면별 상세 명세, UX 원칙, 데이터 모델, API, 알림 전략, 개발 일정을 망라한 완전한 PRD

1. 제품 비전 & 원칙

PRODUCT VISION

“비즈니스 세계의 114”

아무 회사, 아무 부서, 아무 담당자를 — 동의 하에 — 찾아 연락할 수 있는 세상.
오프라인 명함 교환의 신뢰와 예의를 디지털로 그대로 옮긴 서비스.

5가지 디자인 원칙

🔍

1. 3클릭 원칙

검색 → 프로필 확인 → 요청 완료. 어떤 사용 흐름이든 3클릭 이내에 핵심 목표를 달성할 수 있어야 한다.

핵심 UX 원칙
🤝

2. 명함 교환 비유

오프라인에서 명함을 교환하는 것처럼, 서로 동의해야 연락처가 공개된다. 일방적 정보 제공이 아닌 양방향 합의.

신뢰 모델
👑

3. 받는 사람이 왕

연락을 받는 사람이 항상 통제권을 보유한다. 수락/거부/차단 모두 수신자의 자유이며, 어떤 압력도 없어야 한다.

프라이버시
🔒

4. 스팸 제로

구조적으로 원치 않는 연락이 불가능하다. 목적 명시 필수, 승인 전 번호 비공개, 일일 요청 횟수 제한 등 다층 방어.

보안 설계
⚡

5. 1분 온보딩

가입부터 첫 검색까지 60초 이내. 최소한의 정보만 입력하고, 나머지는 나중에 천천히 채울 수 있다.

사용성

2. 사용자 정의 & 핵심 페르소나

MVP에서 집중할 3개 핵심 페르소나. 모든 기능은 이 3명의 관점에서 검증한다.

A

영업 담당자

연락처를 찾는 사람
목표잠재 고객사의 구매 의사결정자를 찾아 미팅을 잡고 싶다
핵심 니즈회사/부서/직책 기반 검색, 연락 요청 시 목적 전달, 승인 여부 실시간 확인
성공 지표요청 승인율 30%+, 주 5건+ 신규 미팅
사용 빈도주 3~5회, 일 5~15건 요청
B

구매/발주 담당자

공급업체를 찾는 사람
목표새로운 공급업체의 영업 담당자를 빠르게 찾아 견적을 받고 싶다
핵심 니즈산업/제품 기반 업체 검색, 회사 인증 여부 확인, 복수 업체 동시 비교
성공 지표신규 업체 미팅까지 3일 이내, 후보 3개사+ 확보
사용 빈도월 1~2회 집중 검색 (프로젝트 기반)
C

수신자

연락 요청을 받는 사람
목표가치 있는 비즈니스 기회만 선별해서 수락하고, 스팸은 원천 차단하고 싶다
핵심 니즈요청자 신원 확인, 목적별 자동 필터, 차단/신고 기능, 알림 커스텀
성공 지표원치 않는 연락 0건, 유용한 비즈니스 연결 월 2건+
사용 빈도주 2~3회 수신함 확인, 즉시 푸시로 응답

3. 핵심 사용자 흐름 (User Flows)

Flow 1

회원가입 & 온보딩 (< 2분)

가입부터 첫 검색까지 2분 이내 완료. 필수 입력은 최소화하고, 나머지는 선택 사항으로 나중에 채울 수 있다.

1

로그인 방식 선택

이메일+비밀번호 / 카카오 / 구글 / 애플 소셜 로그인. 소셜 로그인 시 이름과 이메일 자동 채워짐.

2

기본 정보 입력 (30초)

이름 / 회사명 / 부서 / 직책. 회사명은 자동완성으로 빠르게 입력. 직책은 드롭다운 선택.

3

전화번호 인증 (20초)

전화번호 입력 → SMS 인증번호 6자리 수신 → 자동 입력(안드로이드). 이 번호는 상대방 승인 전에는 절대 노출되지 않는다.

4

명함 사진 OCR (선택, 10초)

명함 사진 촬영 → AI가 이름/회사/부서/직책/이메일 자동 추출 → 사용자 확인. 선택 단계 — 건너뛰기 가능.

5

완료 & 첫 검색 유도

“가입 완료! 지금 바로 찾고 싶은 담당자를 검색해보세요” → 검색바 포커스 + 인기 검색어 표시.

UX 포인트: 소셜 로그인 시 Step 1~2가 합쳐져 실질 3단계. 전화번호 인증 포함 총 소요 시간 60~90초.
Flow 2

연락처 검색 & 요청 (≤ 3클릭)

핵심 흐름. 홈 화면에서 검색하고, 프로필을 확인하고, 연락처를 요청하는 3단계.

1

클릭 1 — 검색

홈 화면 상단 검색바에 “삼성전자 구매팀” 입력. 실시간 자동완성(회사명, 부서명, 이름)이 드롭다운으로 표시. 검색 버튼 클릭 또는 엔터.

2

클릭 2 — 프로필 선택

검색 결과 리스트: 이름 / 회사 / 부서 / 직책 표시. 전화번호는 “***-****-****”로 마스킹. 원하는 사람의 카드를 탭.

3

클릭 3 — 연락처 요청

[연락처 요청] 버튼 클릭 → 목적 선택 (영업/구매/협업/투자/채용 중 택 1) + 한줄 메시지 입력(선택) → [보내기] 클릭.

전송 후: “요청이 전송되었습니다. 상대방이 수락하면 알려드릴게요!” 메시지 표시 + 수신자에게 즉시 푸시 알림 발송.
Flow 3

연락 요청 수신 & 처리

수신자 관점의 흐름. 푸시 알림을 받고, 요청자 정보를 확인하고, 수락 또는 거부를 결정한다.

1

푸시 알림 수신

“김영호 (A식품 영업팀)님이 영업 목적으로 연락처를 요청했습니다” — 탭하면 앱 수신함으로 이동.

2

요청자 프로필 확인

요청자의 이름, 회사, 부서, 직책, 요청 목적, 한줄 메시지를 확인. 요청자의 회사 인증 배지, 가입일, 승인율 정보도 표시.

3

수락 또는 거부

[수락] → 서로의 전화번호가 동시에 공개됨 + 양쪽 알림 발송 + 주소록에 자동 추가.
[거부] → 요청자에게 “거부되었습니다” 알림 (사유 선택 가능: 관심 없음/시기 적절하지 않음/기타).

수락 시 화면:
“연락처가 교환되었습니다!
김영호님의 연락처: 010-1234-5678
지금 바로 전화하시겠습니까?”
[전화하기] [메시지 보내기] [나중에]
거부 시 화면:
“요청을 거부했습니다.”
사유 선택(선택): 관심 없음 / 시기 적절하지 않음 / 기타
[이 사람 차단하기] (선택)
Flow 4

내 연락처(주소록) 관리

수락된 연락처를 관리하는 개인 주소록. 검색, 필터, 메모 추가, 연락 이력 확인이 가능하다.

기능설명
연락처 목록수락된 연락처를 이름/회사/부서별로 정렬. 최근 연락 순, 가나다 순 토글.
검색 & 필터이름, 회사, 부서, 직책으로 검색. 목적별(영업/구매/협업) 필터.
메모 추가각 연락처에 개인 메모 작성 가능 (“2월 미팅 예정”, “견적 요청 중” 등)
연락 이력연결된 날짜, 최초 요청 목적, 메모 히스토리 확인
연락처 내보내기선택한 연락처를 CSV/vCard로 내보내기 (유료 기능)

수신함 비어있음“아직 받은 요청이 없어요. 프로필을 완성하면 요청을 받기 쉬워져요!”

보낸 요청 없음“아직 보낸 요청이 없어요. 누구를 찾고 계세요?”
스켈레톤 UI: 모든 데이터 로딩에 회색 펄스 애니메이션 스켈레톤 적용. 실제 카드 모양과 동일한 레이아웃으로 로딩 시 화면 점프 방지.

♿ 접근성 & 포용적 설계

색약/색맹 대응

색상만으로 정보를 전달하지 않음. 수락/거부는 색상 + 아이콘 + 텍스트 3중 표시. WCAG 2.1 AA 대비율 4.5:1 이상 준수.

스크린 리더

VoiceOver(iOS) / TalkBack(Android) 완전 지원. 모든 인터랙티브 요소에 aria-label 부여. 이미지에 alt 텍스트 필수.

터치 & 모터 접근성

최소 터치 영역 44x44px. 스와이프 대체 버튼 항상 제공. 시간 제한 기능에 연장 옵션 제공.

보안 설계

전화번호 E2E 암호화

전화번호는 AES-256으로 암호화 저장. 수락 시에만 복호화하여 양쪽에 공개. DB 관리자도 평문 확인 불가.

JWT + Refresh Token

Access Token 15분, Refresh Token 7일. httpOnly 쿠키 저장. 로그아웃 시 서버사이드 토큰 무효화.

Rate Limiting

Redis 기반 슬라이딩 윈도우. API별 차등 제한 (인증 5/분, 검색 30/분, 요청 10/분). IP + User ID 복합 키.

인프라 보안

HTTPS 필수. HSTS, CSP 헤더. WAF(Web Application Firewall). DDoS 방어 (Vercel Edge). 보안 감사 로그 90일 보관.

13. 런칭 전략

3단계 런칭 프로세스

Phase 1 Closed Beta — 100명 (4주)

초대 코드 기반 100명 제한. IT/스타트업 업계 종사자 위주. 핵심 흐름(검색 → 요청 → 수락) 검증. 주 2회 사용자 인터뷰. NPS 40+ 달성 시 Phase 2 진입.

Phase 2 Open Beta — 1,000명 (6주)

대기자 명단(Waitlist) 순차 초대. 스케일 테스트 (동시 접속 500명). 알림 시스템 안정성 검증. 결제 기능 라이브 테스트. 승인율 35%+ 달성 시 Phase 3 진입.

Phase 3 정식 런칭 — 무제한

앱 스토어 + 웹 동시 공개. 프레스 릴리즈 + 매체 인터뷰. 런칭 프로모션 (첫 달 Pro 무료). 바이럴 초대 인센티브 (초대 1명당 무료 요청 +5회).
