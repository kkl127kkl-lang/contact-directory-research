제15조 — 개인정보의 수집 및 이용 동의 요건

개인정보보호법 제15조 제1항: 개인정보처리자는 다음 각 호의 어느 하나에 해당하는 경우에만 개인정보를 수집할 수 있다.

호근거우리 서비스 해당
제1호정보주체의 동의를 받은 경우핵심 근거
제2호법률에 특별한 규정 또는 법령상 의무를 준수하기 위해 불가피한 경우해당없음
제3호공공기관이 법령 소관 업무를 수행하기 위해 불가피한 경우해당없음
제4호정보주체와의 계약 체결 및 이행을 위해 불가피한 경우보조 근거
제5호정보주체 또는 그 법정대리인이 의사표시 불능 등해당없음
제6호개인정보처리자의 정당한 이익 (2023년 개정 신설)보조 근거
✅
적합성 판단: 동의 기반 모델은 제15조 제1항 제1호에 구조적으로 완전 부합합니다. 사용자가 스스로 프로필을 등록하고 공개 범위를 설정하는 방식 자체가 "정보주체의 동의"에 해당합니다. 추가로 제4호(계약 이행)와 제6호(정당한 이익)를 보조 근거로 활용할 수 있습니다.

제17조 — 제3자 제공 동의 5가지 고지사항

개인정보를 제3자에게 제공하려면 다음 5가지를 정보주체에게 알리고 동의를 받아야 합니다 (제17조 제2항).

#고지사항우리 서비스 구현 방안
1개인정보를 제공받는 자연락처 열람 요청자 프로필 명시 (회사명, 직책, 요청 사유)
2제공받는 자의 이용 목적"비즈니스 네트워킹/협업 연락" 목적으로 한정 명시
3제공하는 개인정보의 항목이름, 직장, 직책, 이메일, 전화번호 등 사용자가 선택한 항목만
4제공받는 자의 보유 및 이용 기간열람 후 30일, 또는 연결 수락 시 연결 유지 기간
5동의 거부권 및 거부 시 불이익"거부 가능, 거부 시 해당 요청자에게 연락처 미공개" 명시
💡
구현 핵심: 연락처 공개 요청이 들어올 때마다 실시간 팝업으로 5가지 고지 + 승인/거절 버튼을 제공해야 합니다. "포괄 동의"가 아닌 건별 동의 방식이 법적으로 가장 안전합니다.

제22조 — 동의 방법 (필수 vs 선택 분리)

필수 동의

  • 서비스 제공에 반드시 필요한 최소한의 정보
  • 이름, 이메일 (계정 생성용)
  • 소속 회사, 직책 (디렉토리 기본 정보)
  • 거부 시 서비스 이용 불가 고지 필요
  • 제22조 제3항: 필수 동의 거부를 이유로 서비스 제공을 거부하는 것은 적법

선택 동의

  • 서비스 제공에 필수적이지 않은 추가 정보
  • 전화번호, SNS 계정, 포트폴리오 URL
  • 마케팅 목적 이용, 프로필 추천 노출
  • 거부해도 기본 서비스 이용에 제한 없음
  • 제22조 제4항: 선택 동의를 받지 않았다는 이유로 서비스 제공 거부 금지
⚠️
주의: 제22조 제5항에 따라 필수/선택 동의는 구분하여 각각 받아야 하며, 하나의 체크박스로 일괄 동의를 받으면 위반입니다. 또한 선택 동의 항목은 기본값이 "미체크" 상태여야 합니다 (사전 동의 금지).

제37조 — 동의 철회(처리 정지) 요건

정보주체의 권리

  • 언제든지 동의를 철회할 수 있음 (제37조 제1항)
  • 철회 시 지체 없이 수집된 개인정보를 파기 (제21조)
  • 철회가 수집보다 쉬워야 함 (같은 방법, 같은 절차)
  • 앱 내 버튼 1~2회 클릭으로 즉시 철회 가능해야 함
  • 회원 탈퇴와 동의 철회를 구분하여 제공

서비스 구현 요건

  • 계정 삭제: 설정 > 계정 삭제 (5일 유예 후 완전 파기)
  • 프로필 비공개: 즉시 검색 결과에서 제외
  • 특정 항목 철회: 전화번호만 비공개 등 부분 철회
  • 제3자 제공 철회: 이미 제공된 정보 제공받은 자에게 파기 통보
  • 파기 증명: 요청 시 파기 확인서 발급

자본시장법 — 투자중개업 해당 여부 분석

자본시장과 금융투자업에 관한 법률 (자본시장법) 제6조에 따른 "투자중개업" 해당 여부가 핵심 쟁점입니다.

구분"소개""중개""권유"
정의단순히 당사자를 서로 연결해주는 행위거래 성립을 위해 적극적으로 매개하는 행위특정 금융투자상품의 매매를 권고하는 행위
라이선스불필요투자중개업 인가 필요투자권유대행인 등록 필요
우리 서비스해당 — 연락처 제공만비해당 — 거래 개입 없음비해당 — 투자 추천 없음
핵심 판단 기준거래 의사결정에 관여하지 않음거래 조건 협상/조율에 관여수익률/위험 분석 제공
⚠️
경계선 주의: "투자자 매칭", "딜 소싱"과 같은 표현을 서비스 내에서 사용하면 "중개"로 해석될 위험이 있습니다. 마케팅 문구에서 "비즈니스 네트워킹", "연락처 교환"으로 한정해야 합니다.

제6조 — 6가지 적법 처리 근거

#법적 근거적용 가능성설명
(a)동의 (Consent)핵심 근거정보주체가 자신의 개인정보 처리에 명확히 동의한 경우. 우리 서비스의 핵심 모델.
(b)계약 이행 (Contract)보조 근거서비스 이용계약 체결/이행을 위해 필요한 처리. 계정 생성, 프로필 표시 등.
(c)법적 의무 (Legal Obligation)제한적조세, 회계 의무 이행을 위한 결제 정보 보관 등.
(d)중대한 이익 (Vital Interests)해당없음생명/건강 위협 상황에서만 적용.
(e)공적 임무 (Public Task)해당없음공공기관 전용.
(f)정당한 이익 (Legitimate Interests)보조 근거서비스 개선, 사기 방지 등. 반드시 LIA(Legitimate Interest Assessment) 수행 필요.

제7조 — 유효한 동의(Valid Consent) 4대 요건

1. 자유롭게 부여 (Freely Given)

동의 거부 시 불이익이 없어야 함. 서비스 이용과 연결하되 "필수" 최소화. 동의 번들링 금지 (서비스별 개별 동의).

2. 구체적 (Specific)

각 처리 목적별로 별도의 동의. "비즈니스 네트워킹용 프로필 검색 노출" 등 목적을 명확하게 특정.

3. 고지된 (Informed)

컨트롤러 신원, 처리 목적, 수집 항목, 철회 권리, 국외 이전 여부 등을 사전에 명확히 고지.

4. 명확한 의사표시 (Unambiguous)

적극적 행위(체크박스 클릭, 서명 등)로 동의. 사전 체크된 박스, 침묵, 비활동은 동의가 아님.

제37조 — DPO(개인정보보호 책임자) 지정 의무

다음 3가지 중 하나에 해당하면 DPO 지정 의무:

  • (a) 공공기관이 처리하는 경우
  • (b) 핵심 활동이 대규모 정기적/체계적 모니터링인 경우
  • (c) 핵심 활동이 특수 범주 데이터 대규모 처리인 경우
⚠️
판단: 비즈니스 연락처 디렉토리는 (b)에 해당할 가능성이 높습니다. 사용자 프로필 데이터를 체계적으로 처리하므로 DPO 지정을 권장합니다. 초기에는 외부 DPO 서비스(Privasee, DPO-as-a-Service 등) 활용 가능.

제35조 — DPIA(개인정보영향평가) 필요 여부

EDPB 9가지 기준 중 2개 이상 해당 시 DPIA 필수:

  • 해당 평가/점수화 (프로필 매칭 알고리즘)
  • 해당 대규모 데이터 처리
  • 해당 체계적 모니터링 (사용자 활동 추적)
  • 비해당 민감 데이터 처리
  • 비해당 자동화된 의사결정
  • 비해당 취약 계층 데이터
📋
결론: 최소 2~3개 기준 해당으로 DPIA 수행 필수. 서비스 출시 전 완료해야 합니다.

한국-EU 적정성 결정 (2022년 12월) 활용

🇮🇷
2022년 12월 17일 EU 집행위원회가 한국을 적정성 결정(Adequacy Decision) 국가로 인정했습니다. 이로 인해:
  • EU-한국 간 개인정보 이전에 별도 안전장치(SCC 등) 없이 자유로운 이전 가능
  • 한국 PIPA가 EU GDPR과 "본질적으로 동등한" 수준으로 인정
  • 한국 서비스가 EU 사용자 데이터를 한국 서버에 저장 가능
  • 단, 적정성 결정에는 "보충 규칙"이 포함되어 있어 이를 준수해야 함
  • 4년마다 재검토 (2026년 말 첫 재검토 예정)

제17조 — 삭제권 (Right to Erasure)

  • 정보주체 요청 시 지체 없이 삭제
  • 제3자에게 공개한 경우 해당 제3자에게도 삭제 통보
  • 기술적으로 불가능한 경우 처리 제한으로 대체
  • 구현: 계정 삭제 시 30일 유예 + 완전 삭제. 백업에서도 90일 내 제거.
  • 삭제 요청 로그 5년 보관 (증빙용)

제20조 — 이동권 (Right to Data Portability)

  • 정보주체가 제공한 데이터를 구조화된 기계 판독 가능 형식(JSON/CSV)으로 제공
  • 다른 서비스로 직접 전송 요청 가능
  • 동의 또는 계약 근거로 자동 처리된 데이터에만 적용
  • 구현: 설정 > 내 데이터 다운로드 (프로필, 연락처, 활동 이력)
  • 형식: JSON + CSV 동시 제공, 30일 내 처리
