2. 사용자 여정 (User Journey) 4개

Journey 1: 연락 요청자 여정 (검색 → 연결)

페르소나: 김영호(영업팀장) — B2B 잠재 고객의 IT 담당자를 찾아 미팅을 요청하는 전체 흐름

1단계: 가입 & 회사 인증 감정: 기대 +7

이메일/소셜 가입 → 회사 이메일 도메인 인증 → 회사명/직책 입력 → 프로필 사진 업로드. 소요 시간 3~5분. 회사 인증 완료 시 "인증된 프로필" 배지 발급.

2단계: 프로필 최적화 감정: 보통 +5

전문 분야 태그 설정(최대 10개), "받고 싶은 연락" 유형 선택 (영업 제안/파트너십/투자/채용 등), 공개 범위 설정 (전체/인증사용자/승인자), 자기소개 작성(500자). 프로필 완성도 점수(%) 표시.

3단계: 검색 & 탐색 감정: 흥분 +8

산업/직무/회사 규모/지역 필터 조합 검색. 검색 결과에서 프로필 미리보기(이름/회사/직책/태그). "이 사람에게 연락 가능한 유형" 아이콘 표시. 유료 사용자: 고급 필터(회사 매출 규모, 투자 라운드, 기술 스택). 검색당 평균 15~30초 소요 목표.

4단계: 연락 요청 작성 감정: 긴장 +4

상대방이 허용한 연락 유형 중 선택. 요청 목적 300자 이내 작성 (템플릿 제공). 자신의 프로필 요약 자동 첨부. "보내기" 클릭. 무료: 월 5건, Pro: 월 50건, Enterprise: 무제한. 잔여 크레딧 실시간 표시.

5단계: 승인 대기 감정: 불안 +2

상대방 승인 대기 (평균 24~48시간). 진행 상태 표시: 발송됨 → 읽음 → 승인/거절. 48시간 미응답 시 자동 리마인더 (1회). 거절 시 "거절 사유 선택" (선택사항): 관심 없음/시기 부적절/정보 부족.

6단계: 대화 시작 감정: 만족 +9

승인 알림 수신 (푸시/이메일). 1:1 채팅방 자동 생성. 연락처 교환 (이메일/전화, 단계별 공개). 대화 후 "연결 유지/종료" 선택. 성공적 연결 시 양측 프로필에 "연결 완료" 카운트 증가.

Journey 2: 요청 수신자 여정 (승인/거절)

페르소나: 정민수(시니어 개발자) — 스팸 없이 원하는 연락만 받는 흐름

1단계: 알림 수신 감정: 호기심 +6

푸시 알림: "[회사명]의 [이름]님이 [목적] 관련 연락을 요청합니다". 알림 탭에 요약: 요청자 프로필 카드 + 요청 목적 미리보기 (100자). 일일 요약 이메일 (설정 가능): 미응답 요청 N건.

2단계: 요청 상세 확인 감정: 평가 +5

요청자의 전체 프로필 열람 (회사/직책/경력/공통 연결). 요청 목적 전문 확인 (300자). 요청자의 프로필 신뢰 점수 확인: 회사 인증 여부, 프로필 완성도, 기존 연결 수, 응답률. "공통 연결 N명" 표시 (신뢰도 향상).

3단계: 의사결정 & 응답 감정: 통제감 +8

[승인]: 대화방 열림 + 공개할 연락처 범위 선택. [나중에]: 7일 후 다시 알림. [거절]: 사유 선택(선택) + 차단 옵션. [신고]: 스팸/사칭/부적절 내용 신고. 3초 내 빠른 응답 가능한 스와이프 UI (모바일).

4단계: 연결 후 관리 감정: 안도 +7

대화 후 상대방 평가 (1~5점 + 한줄평). 연결 유지/해제 선택. 차단 설정 (특정 사용자/회사 단위). "받고 싶은 연락" 설정 미세 조정. 연결 이력 대시보드에서 통계 확인.

Journey 3: 무료 → 유료 전환 여정

페르소나: 이수진(스타트업 CEO) — 무료에서 시작해 유료로 자연스럽게 전환되는 흐름

1단계: 무료 체험 (가입 후 1~2주) 감정: 탐색 +6

무료 월 5건 연락 요청 사용. 기본 검색 필터 사용. 프로필 등록 완료 → 인바운드 연락 1~2건 수신. "이 서비스가 유용하다" 인식 형성. 프로필 조회수/검색 노출 통계 확인.

2단계: 한계 인식 (2~3주) 감정: 아쉬움 +3

월 5건 크레딧 소진. 고급 필터(투자 단계별 검색) 잠금 화면 노출. "이 프로필을 보려면 Pro 플랜이 필요합니다" — 블러 처리된 결과 표시. 검색 결과 목록에서 Pro 전용 프로필 강조 (잠금 아이콘).

3단계: 전환 트리거 (3~4주) 감정: 결심 +7

"지난주 프로필 조회 12회, 연락 요청 대기 3건" 이메일 발송. 7일 무료 Pro 체험 제안 (인앱 배너). 유사 사용자 성공 사례: "같은 업종 CEO가 Pro 사용 후 투자자 3명과 미팅 성사". 연간 결제 20% 할인 프로모션.

4단계: 결제 & 활성화 감정: 확신 +8

Pro 플랜 월 49,000원 결제 (카드/법인카드). 즉시 고급 필터 해제 + 월 50건 연락 요청. "Pro 배지" 프로필에 표시 (신뢰도 향상). 결제 후 30일 내 만족하지 않으면 전액 환불 보장.

Journey 4: 기업 관리자 여정 (팀 관리)

페르소나: 최은정(대기업 구매 담당) — 팀 단위로 서비스를 도입하고 관리하는 흐름

1단계: Enterprise 문의 & 도입 감정: 신중 +5

Enterprise 플랜 문의 → 세일즈팀 데모 미팅. 회사 도메인 일괄 인증. 보안 검토: SSO 연동, 데이터 처리 동의서, SLA 확인. 관리자 계정 생성 + 초기 설정 가이드 제공.

2단계: 팀원 초대 & 온보딩 감정: 기대 +6

CSV/이메일 일괄 초대 (최대 500명). 팀별 역할 부여: 관리자/정규/제한. 팀 전용 온보딩 가이드 + 사용 교육 영상 제공. 회사 프로필 페이지 자동 생성 (로고/소개/채용 정보).

3단계: 팀 활동 모니터링 감정: 관리 +7

관리자 대시보드: 팀원별 사용량(검색/요청/연결), 월별 사용 추이 차트, 잔여 크레딧, 비용 요약. 팀 전체 연결 네트워크 맵 (시각화). 이상 사용 알림 (과다 요청, 신고 접수). 월간 사용 리포트 자동 이메일.

4단계: 갱신 & 확장 감정: 확장 +8

연간 계약 갱신 시 ROI 리포트 자동 생성: 총 연결 수, 평균 응답 시간, 비즈니스 성과 추정. 추가 팀(영업팀/마케팅팀) 확장 제안. API 연동(CRM/ERP) 추가 옵션. 전담 CSM(Customer Success Manager) 배정.

4. "동의 기반" UX 설계

연락 요청 UI — 승인률을 높이는 5가지 원칙

원칙 1: 투명한 목적 제시

요청 목적을 5가지 유형 중 선택하도록 강제 (영업 제안/파트너십/투자/채용/네트워킹). "기타" 옵션은 제공하지 않음 — 모호한 목적은 거절률을 60% 높임. 수신자가 목적을 한눈에 파악할 수 있도록 아이콘+색상으로 시각 구분.

원칙 2: 상호 프로필 노출

요청자의 프로필이 수신자에게 100% 공개됨 (비공개 요청 불가). 이름/회사/직책/프로필 사진/경력 요약/공통 연결 수. "상대방도 나를 볼 수 있다"는 심리적 안전감이 승인률을 2배 높임.

원칙 3: 연락 비용 부과

연락 요청에 크레딧 소모 (무료 1건/유료 1건). 비용이 있으면 스팸성 대량 발송 억제. 크레딧 잔량 실시간 표시로 "아끼게" 유도. 승인되면 크레딧 반환 없음, 거절 시 50% 반환.

원칙 4: 응답 시간 제한

수신자가 72시간 내 응답하지 않으면 자동 만료 + 크레딧 100% 환불. 24시간 후 리마인더 1회 자동 발송. 만료된 요청은 30일 후 재요청 가능. 빠른 응답자에게 "빠른 응답 배지" 부여.

원칙 5: 거절의 부담 최소화

거절 시 요청자에게 "현재 새로운 연결을 받지 않습니다"로 표시 (개인적 거절 느낌 방지). 거절 사유는 선택사항이며 익명 처리. "나중에" 옵션으로 즉각적 의사결정 부담 제거. 거절해도 상대 프로필에 부정적 영향 없음.

스팸 방지 8가지 메커니즘

1. 크레딧 기반 요청 제한

무료 월 5건, Pro 월 50건, Enterprise 무제한(단 일 100건 제한). 크레딧 소진 시 추가 구매 불가(월 리셋만 가능). 대량 발송의 경제적 비용 부과.

2. 프로필 완성도 게이트

프로필 완성도 70% 미만 사용자는 연락 요청 불가. 필수 항목: 실명, 회사, 직책, 프로필 사진, 자기소개. 불완전 프로필 = 스팸 계정 가능성 높음.

3. 회사 이메일 인증 필수

개인 이메일(gmail/naver) 사용자는 검색만 가능, 연락 요청 불가. 회사 이메일 인증 시 스팸 계정 생성 비용 급증. 1개 도메인당 최대 등록 수 제한.

4. 거절률 기반 제재

최근 30일 거절률 70% 초과 시 경고 → 80% 초과 시 요청 수 50% 감소 → 90% 초과 시 7일 요청 차단. 신고 3회 이상 시 즉시 리뷰 → 영구 차단 가능.

5. 메시지 내용 AI 필터

요청 메시지 발송 전 AI 자동 검토. 감지 대상: 욕설, 성희롱, 피싱 URL, 동일 메시지 복사붙여넣기 패턴, 과도한 이모지. 의심 메시지: 발송 지연 + 수동 리뷰 큐.

6. 동일 대상 재요청 제한

거절된 대상에게 30일 내 재요청 불가. 만료(미응답)된 요청은 7일 후 재요청 가능(1회). 차단된 대상에게는 영구 요청 불가. 우회 시도(새 계정) 탐지.

7. 사용자 신고 시스템

Cold Start 해결 5단계

1단계: 시드 사용자 확보 (출시 전 2개월) 목표: 500명

스타트업 커뮤니티(디캠프/마루180) 파트너십으로 초기 사용자 유치. 초기 사용자 전원 "Founding Member" 배지 + Pro 기능 3개월 무료. VC/엔젤 투자자 50명 직접 초대 (대표가 직접 이메일). 테크 밋업/컨퍼런스에서 명함 스캔 이벤트.

2단계: 공급 측 집중 (출시 후 1~2개월) 목표: 2,000명

"받고 싶은 연락만 받는다"는 가치 제안으로 수요자(연락 받는 사람) 먼저 확보. 시니어 개발자/CTO/부서장 등 "연락 받고 싶은 사람"이 먼저 프로필 등록. 영업 제안 수신 설정 시 "관심 있는 제품 카테고리" 등록 → 매칭 가능 데이터 축적.

3단계: 수요 측 유입 (출시 후 2~4개월) 목표: 5,000명

"검색 가능한 의사결정자 N명" 마케팅으로 영업 담당자/VC 유입. 콘텐츠 마케팅: "B2B 영업 담당자가 알아야 할 효율적 네트워킹 법" 블로그/뉴스레터. 리퍼럴 프로그램: 초대한 친구가 가입 시 양측 모두 크레딧 +5건.

4단계: 산업별 클러스터 형성 (출시 후 4~6개월) 목표: 10,000명

IT/핀테크/바이오 등 특정 산업에 집중 공략 → 산업 내 네트워크 밀도 확보. 산업별 "커뮤니티 리더" 임명 (인플루언서 10명) → 해당 산업 사용자 유치 위임. 산업별 네트워킹 이벤트 (온라인/오프라인) 월 1회 개최.

5단계: 바이럴 루프 가동 (출시 후 6개월+) 목표: 자생 성장

연결 성공 시 "이 서비스에서 [이름]님과 연결되었습니다" 소셜 공유 유도. 기업 관리자가 팀원 초대 → B2B 바이럴 (팀 단위 도입). "연결 추천" 알림으로 비활성 사용자 재활성화. 월간 "네트워크 인사이트" 이메일 (프로필 조회수, 산업 트렌드).

DB 설계 원칙:

1. Soft Delete: User, Company, Conversation에 deletedAt 필드 적용 (데이터 영구 삭제 방지)
2. JSON 필드: 유연한 스키마가 필요한 곳(태그, 경력, 필터 조건)에 JSON 사용, 인덱싱은 GIN으로
3. UUID PK: 모든 테이블 UUID v4 사용 (순차 ID 노출 방지, 분산 DB 호환)
4. 인덱스: email(UNIQUE), companyId(FK), senderId+status(복합), userId+createdAt(복합), query+userId(검색 로그)
5. GDPR 준수: 계정 삭제 시 개인 정보 익명화 처리 (이름/이메일 해시 대체), 메시지 내용 삭제, 연결 이력 익명화
