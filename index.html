"모든 회사의 모든 부서 담당자를 합법적으로 찾아, 동의하에 연락하는" 세계 최초의 플랫폼

핵심 발견

"특정 회사의 특정 부서 담당자를 합법적으로 찾아 동의하에 연락하는 플랫폼"은 현재 전 세계에 없습니다.

기존 업체(ZoomInfo, Apollo 등)는 동의 없이 데이터를 긁어서 파는 모델이라 프라이버시 규제 강화에 취약하고, 이 구조적 약점이 가장 큰 기회입니다.

서비스 핵심 메커니즘 — "명함 교환" 방식

이 서비스는 LinkedIn(SNS)이 아닙니다. LinkedIn은 "나를 보여주는 곳"이고, 이 서비스는 "상대를 찾는 곳"입니다. LinkedIn에서 "삼성전자 구매팀 김과장 직통번호"를 알 수 있나요? 절대 못합니다. 그게 이 사업의 핵심입니다.

요청 목적별 수락/거부 규칙

요청 목적받는 사람 선택권예상 수락률이유
영업/세일즈거부 가능20~40%원치 않는 영업 = 스팸. 받는 사람이 통제권 보유
구매/발주거부 가능80~95%돈 주겠다는 사람을 거부할 이유가 거의 없음
파트너십/협업거부 가능40~60%상호 판단에 따라 결정
투자 관련거부 가능60~80%IR팀은 투자자 연락을 대부분 환영
채용거부 가능30~50%이직 의사 없으면 거부

왜 이게 혁명적인가?

영업하는 쪽: 수락한 사람 = 관심 있는 사람. 거부당하면 시간 낭비 없음 → 전환율 극대화

받는 쪽: 내가 원하는 연락만 받음 → 스팸 제로

핵심: "내 명함을 먼저 주고, 상대가 허락해야 받는다" — 전 세계 어디에도 없는 방식

LinkedIn과 완전히 다른 이유

영업 시나리오 (거부 가능)

A식품 영업팀 → B프랜차이즈 구매팀

1. A식품 박대리가 "B프랜차이즈 > 구매팀 > 식자재 담당" 검색

2. 김과장 프로필 발견 (번호 가려짐)

3. [연락처 요청] → 목적: 영업 / 품목: 소스류

4. 김과장에게 알림: "A식품 박대리가 영업 목적으로 연락처를 요청합니다"

→ 김과장: "소스는 충분해" → 거부 → 끝

→ 또는: "새 소스 찾고 있었는데!" → 수락 → 서로 번호 공개

구매 시나리오 (거의 항상 수락)

B프랜차이즈 구매팀 → A식품 영업팀

1. 김과장이 "소스 제조사 > 영업팀 > 담당자" 검색

2. A식품 박대리 발견

3. [연락처 요청] → 목적: 구매 문의

4. 박대리에게 알림: "B프랜차이즈 김과장이 구매 목적으로 요청"

→ 박대리: 구매 제안을 거부할 영업사원은 없음 → 즉시 수락

→ 서로 번호 공개 → 거래 시작
